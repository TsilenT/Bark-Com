# Commit Message Conventions

To ensure our automated patch notes generator correctly categorizes your changes, please start your commit messages with one of the following keywords.

## 🚀 Features & Content
Start with: `feat`, `new`, `add`, `implement`
*   `feat: Add new sniper class`
*   `new: Create level 5 map`
*   `add: Inventory system`
*   `implement: Fog of war`

## 🐛 Bug Fixes
Start with: `fix`, `bug`, `repair`, `resolve`, `hotfix`
*   `fix: Game crash on load`
*   `bug: Unit stuck in wall`
*   `hotfix: Critical save corruption`

## 🎨 Visuals & Polish
Start with: `style`, `ui`, `visual`, `art`, `shader`, `polish`
*   `style: Update button colors`
*   `ui: New health bar design`
*   `visual: Add particles to explosion`
*   `shader: Water reflection improvement`

## 🔧 Tech & Refactor
Start with: `refactor`, `perf`, `optim`, `clean`, `test`, `ci`, `chore`
*   `refactor: Move movement logic to separate class`
*   `perf: Optimize pathfinding`
*   `ci: Update github actions`
*   `test: Add unit tests for combat`

## 📄 Documentation
Start with: `doc`, `readme`, `comment`
*   `doc: Update getting started guide`
*   `readme: Fix typo in instructions`

## 📝 Other Changes
Any commit not matching the above will be placed in "Other Changes".
